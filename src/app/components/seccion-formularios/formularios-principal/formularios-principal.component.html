Formulario template: Toda la configuracion y la validaciones se hacen del lado del html

Si hacemos uso de ngModel: debemos usar la propiedad name en nuestra etiqueta html.
Al hacer uso de ngModel automaticamente se a単aden ciertas clases a nuestra caja de texto

  • ng-untouched / ng-touched: Detecta si el usuario hizo o no click en la caja de texto
  • ng-pristine / ng-dirty : Detecta si el usuario introducio o no algun caracter en la caja de texto
  • ng-valid ng-invalid : Detecta si es valido el contenido del campo

Con las clases mencionadas podemos usarlas, para darle los estilos apropiados a los determinados eventos y validaciones

Las propiedades de validacion pueden ser usadas como punto de referencia para las validaciones de angular, usando minlength para angular no sera valido hasta que esta regla se cumpla

Con (ngSubmit)="nombreFuncion()" en la etiqueta del formulario, podemos decirle a que metodo ha de dirigirse al hacer el posteo de informacion.

Con #nombreFormulario="ngForm"en la etiqueta del formulario, podemos usar el formulario como un objeto y mandarselo como parametro en la funcion ngSubmit #formularioTemplate="ngForm" (ngSubmit)="guardarFormulario(formularioTemplate)" novalidate>

De esta manera obtenemos todas las propiedades del formulario, como el estado del formulario, propiedades de entrada, etc..

Lo correcto es usar la clase ngForm para darle el tipo al parametro den entrada de la funcion que recoje el posteo de informacion

Si queremos setear cierta informacion ya en el formulario con los valores de algun objeto podemos usar [ngModel]="objeto.propiedad"
Esto nor permite setear los valores del formulario con los de un objeto , cuando hagamos post, cambiaran los valores del formulario pero no los del objeto. para que eso suceda debemos usar el banana in box , entonces seria totalmente bidirecional la alteracion de datos.

Formulario Reactivo: Toda la configuracion la logica esta al lado del componentes, las validaciones son mas robustas

La proximacion por data nos da mayor control a la hora de hacer validaciones

Para poder hacer uso de los controles y las validaciones, debemos importar los modulos import { FormGroup, FormControl, Validators } from '@angular/forms';

Debemos a単adirlos tambien en el app.module.ts en la seccion de Imports FormsModule, ReactiveFormsModule,

En la etiqueta del formulario agregamos [formGroup]="formulario" donde formulario equivale al nombre con el que hemos declarado nuestro formulario en el .ts formulario : FormGroup; De esta forma relacionamos el formulario html con el formulario del ts

A単adimos (ngSubmit)="metodoRecepcionPosteo()" a la etiqueta formulario para poder recivir los datos en la funcion metodoRecepcionPosteo()

Para enlazar los inputs de nuestro formulario html con los inputs definidos dentro del formulario(formGroup) del ts. debemos a単adir formControlName="nombre" a cada input de esta forma antendera a las especificaciones de requisitos que se definan para cada campo en el .ts .

Ahora si modificamos la definicion de los formControl que estan dentro del formulario del .ts 'nombre' : new FormControl('',Validators.required), estamos creando el control requerido para ese input, que de no ser valido a su vez inhabilita el estado del formulario
